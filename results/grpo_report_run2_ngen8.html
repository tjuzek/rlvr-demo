Why the absolute gain is modest: sparse rewards at low baseline

The baseline model solves only 2.7% of problems. For GRPO to learn, each group of G completions needs at least one passing sample — otherwise every sample gets reward 0, the within-group advantage is 0, and the gradient contribution is 0.

With G = 8 generations per prompt and baseline pass rate p = 0.027, the fraction of groups that are entirely zero-reward is (1 − p)G = (1 − 0.027)8 ≈ 80.2%.

In other words: ~80% of optimizer steps carry no learning signal at all. Only the remaining 20% contribute gradient. The run is doing real RL, but on a fraction of the compute it looks like it's doing.

Counter-moves: more generations per prompt (G = 8 → 80% zero-reward · G = 16 → 64%), filter training set to problems the base model solves ≥ sometimes (cheapest, biggest lift), or warm-start with SFT to raise the baseline first. Tulu 3 does all three.
